Why passwords matters

Once in a while data leaks occurs, something that happens when systems or companies are not secure enough or manage to follow up on the latest vulnerabilities. Password has been used since the earliest days of computing. Although the development of security standards is constantly changing and improving, the principles and use of passwords remain the same. To minimize the attack surface of the various services you use, you should have unique passwords.

Documentation

To get started, review the help page using:

cpm --help

Available options:

-s, --site TEXT -u, --username TEXT -l, --length INTEGER -d, --digest [SHA1|SHA224|SHA256|SHA384|SHA512]

There are several options for creating passwords. One can specify the url or site name of the web page the password is intended to be used on. It is also possible to specify a desired password length.

Examples

cpm -s google -l 16 cpm -s github -u gh -l 8 -d SHA512
